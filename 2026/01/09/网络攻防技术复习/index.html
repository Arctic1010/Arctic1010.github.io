网络攻防技术-课程部分复习

CH1 概述

  1. 网络攻击分类：

    • 标准：

      • 互斥性（分类标准不应重叠）
      • 完备性（覆盖所有可能攻击）
      • 非二义性（类别划分清晰）
      • 可重复性（同一样本多次分类结果一致）
      • 可接受性（符合逻辑和直觉）
      • 实用性（可用于深入研究和调查）

    • 类别：

      • 物理攻击：通过实际接触被攻击的主机实施的各种攻击。
      • 主动攻击：攻击者利用 Web、FTP、Telnet 等开放网络服务对目标实施的各种攻击。
      • 被动攻击：攻击者利用客户端应用程序漏洞或用户弱点，对目标实施的各种攻击。
      • 中间人攻击：攻击者处于被攻击主机的某个网络应用的中间人位置，进行窃听、破坏或篡改等攻击。

  2. 攻击步骤：信息收集，权限提取，安装后门，扩大影响，清除痕迹

    • 信息收集：尽可能多地收集目标的相关信息，为后续的攻击建立基础。
    • 权限提取：获取目标系统的 rwx 等权限。
    • 安装后门：向目标系统安装后门程序，更加隐蔽、方便地进行操控
    • 扩大影响：内网穿透，对目标所属网络的其它主机进行攻击。
    • 清除痕迹：清除攻击的痕迹，以尽可能长久地对目标进行控制。

  3. 物理攻击及社会工程学：

    • 物理攻击定义：通过各种技术手段绕开物理安全防护体系，从而进入受保护的设施场所或设备资源内，获取或破坏信息系统物理媒体中受保护信息的攻击方式。
      • 如何防范？门禁安全，笔记本防盗，桌面重要资料，密码，U 盘等保存好。离开时锁电脑。
    • 社会工程学：利用人类的愚蠢，操纵他人执行预期的动作或泄露机密信息的学问。
      • 如何防范？进行安全意识和操作规程培训，注意隐私保护。

CH2 信息收集技术

  1. 公开信息收集：

    定义：黑客为了更加有效地实施攻击而在攻击前或攻击过程中对目标的所有探测活动。

    内容：域名和 IP 地址、防火墙、内部网络结构、操作系统类型、端口开放情况、敏感文件和目录等。

    分类：主动（将流量流经网站的行为），被动（利用第三方服务如搜索引擎）。公开信息收集、网络扫描、漏洞扫描、拓扑探测。

    必要性：在攻防对抗中，对敌方信息的掌握是关键。攻击者具有先手优势，防御者可以发起反击。

    方法：利用 Web 服务、DNS 服务、WhoIs 服务、搜索引擎服务等

  2. 网络扫描的类型：主机、端口、系统类型扫描

    主机扫描：Ping（ICMP），异常 IP 包头，无效字段值，错误数据分片。

    端口扫描：

    • SYN 扫描：半连接方式。发送 TCP-SYN 数据包，收到 SYN-ACK 表示目标端口开放，收到 RST 表示端口关闭，无响应表示主机不活跃或包被防火墙拦截。收到 SYN-ACK 后，发送 RST 避免建立完整连接。
      • 优点：一般不会被目标主机的应用所记录，主要用于躲避防火墙的检测
      • 防御：配置防火墙规则，设置 IDS 检测异常 SYN 包，关闭不必要的端口和服务。
    • FIN、Xmas（FIN+URG+PUSH）、Null：关闭的端口返回 RST。优点为隐蔽性强，缺点为依赖 TCP/IP 栈实现。
    • ACK：收到 RST 表示端口未被防火墙过滤，无回复或收到 ICMP 错误，说明可能被过滤。用于防火墙规则探测或网络拓扑推断。

    系统类型扫描：

    • 根据系统不同，可能默认打开一些特定的端口。
    • 应用程序的 Banner。
    • TCP/IP 协议栈指纹。

  3. 漏洞扫描：

    目的：使用漏洞扫描程序对系统或应用程序的漏洞进行扫描，以找到可以被黑客利用的入口。

    原理：对目标系统进行信息收集，结合漏洞特征库或受控的探测请求，判断系统是否存在已知安全漏洞或配置缺陷。

    组件：被动式策略（基于主机）与主动式策略（基于网络）。

    方法：直接测试、推断、带凭证的测试。

    • 直接测试：利用漏洞特点发现系统漏洞的方法。攻击性较强，效率较低，能够准确判断但是适用性较窄。
    • 推断：通过系统版本、程序行为分析、堆栈分析、时序分析等方法，间接寻找漏洞存在的证据。
    • 凭证测试：让漏洞扫描器登录系统，从内部获取更全面、更准确的安全状态信息。

  4. 网络拓扑探测：

    • 拓扑探测（Traceroute、SNMP）
    • 网络设备识别（搜索引擎、Banner）
    • 网络实体 IP 地理位置定位（查询信息、网络时延测量）

CH3 口令攻击

  1. 口令的定义及作用：

    • 定义：口令是由用户预先设定并仅由用户知晓的一组字符，用于在身份鉴别过程中证明用户身份的一种秘密认证信息。

    • 作用：身份认证。

  2. 针对口令强度的攻击

    • 弱口令与强口令（不容易被发现规律、有足够的长度、在某时间段内很难破解）。
    • 攻击方式：枚举、字典、组合、撞库、彩虹表

  3. 针对口令存储的攻击

    • Linux 将密码存储在 /etc/shadow，Windows 存储在 SAM 文件里。
    • 密码验证：存储哈希值和盐以防止彩虹表攻击。
    • SAM 文件在操作系统运行期间被 system 账号锁定，注册表备份被隐藏，保存时被压缩。使用 NTLM 协议加密。
    • 口令获取方法：读取存储、内存，破解 SAM。

  4. 针对口令传输的攻击

    • 嗅探攻击：同局域网下的 FTP 嗅探，802.3 以太网使用广播信道。
    • 键盘记录：修改硬件接口、监视操作系统的键盘输入接口。
    • 网络钓鱼：使用欺骗性的电子邮件和 Web 站点。
    • 重放攻击：攻击者记录下当前的通讯流量，以后在适当的时候重发给通讯的某一方。

  5. 如何防范：选择安全密码、防止口令猜测（使用 NTFS 格式的硬盘，正确设置账户，禁止不需要的服务和端口）、设置安全策略、定期改变口令。

CH4 软件漏洞（一）缓冲区溢出

  1. 漏洞的定义：指信息系统在设计上、实现上出现的可以被攻击者利用的错误、缺陷和疏漏。

  2. 典型漏洞类型：栈溢出、堆溢出、格式化字符串、整型溢出、释放再使用。

  3. 程序在内存中的存放形式：

    地址 数据 存放
    内存高地址 argv、envp 命令行参数及环境变量
    stack 栈
    heap 堆
    .bss 未初始化全局变量或静态变量
    .data 初始化全局变量或静态变量
    内存低地址 .text 程序指令和只读数据
  • 代码段：.text 存放着机器码与只读数据，可执行指令就是从这里获得的，写操作会导致段错误。
  • 数据段：包括已初始化变量的数据段 .data 和未初始化变量的数据段 .bss。在编译时分配。
  • 堆：位于 bss 段上方。由应用程序决定扩张和缩减：分配由 malloc、new 等实时分配函数来实现；释放由 free、delete 来实现。
  • 栈：存储函数调用时的临时信息的结构，如传参、返回地址、局部变量等。由编译器在需要的时候分配，不需要的时候清除。FILO。栈底在高地址，反着长的，也就是说栈帧往低地址方向延伸。函数栈帧大小不等。

  1. 栈溢出漏洞利用原理：

    • 栈俩寄存器：SP 和 BP。
    • 函数被调用时，栈先后压入：参数、返回地址（IP）、pre BP、局部变量。
      • push eip; call; push ebp; mov ebp, esp;
    • 这里溢出可以造成返回地址的伪造，让其指向 shellcode 或者其他 gadget 等，造成攻击。

  2. 溢出攻击的基本流程：确定溢出点位置、覆盖控制地址、覆盖异常处理结构、跳转地址确定、ShellCode 的定位和跳转。

  3. 漏洞利用保护机制：

    • DEP（数据执行保护）Data Execution Protect：防止恶意代码在非可执行内存中执行。
    • ASLR（地址空间布局随机化）Address Space Layout Randomness：随机化进程中各种关键数据区域，让固定地址变成难以预测的随机地址。
    • Canary（金丝雀）：调用函数时，将小随机数放在栈上，返回前检查其是否被修改。
    • SEHOP（结构化异常处理重写保护）Structured Exception Handling Overwrite Protection：防止攻击者覆盖 SEH 链条。
      • SEH What？是 Windows 提供的异常处理机制。当程序发生异常时，系统会沿着 SEH 链表查找异常处理函数。SEHOP 通过验证链的完整性，阻止被篡改的异常处理函数被执行。
    • CFI（控制流完整性）：程序的执行路径必须严格符合预先定义的合法控制流图。
    • ROP、JOP：通过篡改返回地址，把程序执行“拼接”成攻击者想要的指令序列。
    • 程序签名、沙箱、虚拟机、容器、打补丁。

CH5 软件漏洞（二）ShellCode 编写

  1. ShellCode What？

    • 定义：ShellCode 就是一段能够完成一定功能、可直接由计算机执行的机器代码，通常以十六进制的形式存在。

    • 作用：可以是非攻击性的，如完成一定功能；可以是攻击性的，如发起反向连接、下载木马、删除重要文件等。

    • Windows 函数调用原理：加载动态链接库、使用堆栈进行函数传递、调用函数地址。

    • 局限性：注入时不能有 \x00。

  2. 如何编写 ShellCode？（Lab ShellCode）

    1. 要执行 execve，弄清楚调用形式 execve(const char *filename, const char *argv[], const char *envp[]); Linux x86 系统调用约定：使用 ebx、ecx、edx 依次代表这三个参数。所有数据都在栈内构造。（cdecl 则全都在栈上）

    2. 构造环境变量数组。以 0 结尾。如果单个条目字符数量不能是 4 的倍数，使用 shr 而不是 \x00。存储数组首地址（栈顶）到 edx。

    3. 构造文件名。存储字符串首地址（栈顶）到 ebx。

    4. 假设没有参数，eax 为 0，则 push eax，push ebx。存储数组首地址（栈顶）到 ecx。

    5. xor eax eax，mov al, 0x0b，int 0x80。调用系统调用编号为 0x0b 的函数，即 execve。

CH6 软件漏洞（三）环境变量攻击

  1. 环境变量是什么？一组动态的定义值，操作系统运行环境的一部分，影响正在运行进程的行为方式。

    • 当执行一个程序时，如果没有提供完整的路径，shell 进程将使用 PATH 环境变量来找到程序的位置。

  2. 进程获取环境变量的方式：

    • 如果使用 fork() 创建了一个新进程，则子进程将继承其父进程的环境变量。
    • 如果进程使用 execve() 启动一个新程序，在此场景中内存空间被覆盖，所有旧环境变量将丢失。

  3. 环境变量在内存的位置：

    在 C main 函数的传参里，envp 在 argc、argv 的后面。事实上有两个指针 *envp 和 *environ（全局变量）指向同一个位置（envp[0]）。当对环境变量进行更改时，存储环境变量的位置会移动到堆里，此时会改动 environ 指针，envp 指针保持不变。

  4. 人们经常认为的 Shell 命令变量和环境变量是相同的说法是不对的。

    • 环境变量会被子进程继承，Shell 命令变量只在当前 Shell 内有效。
    • 当 shell 程序启动时，它会将环境变量复制到自己的 shell 变量中。对 shell 变量所做的更改将不会反映出在环境变量上。

  5. /proc 是 Linux 中的一个虚拟文件系统。它包含每个进程的一个目录，使用进程 ID 作为目录的名称。每个进程目录都有一个 environ 文件，表示进程的环境。/proc/932/environ 则是 ID 为 932 的进程的环境变量。

    • 当你在 bash 中运行 env 时，env 不是在 bash 进程里运行的，而是 bash fork 出来的子进程。
      因此，env 打印的是 它所继承到的环境变量，也就是 bash 传给它的那份环境变量副本，而不是 bash 自己内部维护的全部变量。

  6. 由于用户可以设置环境变量，因此它们将成为 Set-UID 程序的攻击面的一部分。

    • **Set-UID What？**允许用户以程序所有者的权限执行程序。

    • RUID 和 EUID 分别是启动该进程的真实用户和进程权限所属的用户。

      • Set-UID 程序文件属主为 root，alice 启动程序，RUID = alice，EUID = root，程序以 root 权限运行。

      • 未 Set-UID 程序文件属主为 root，alice 启动程序，RUID = alice，EUID = alice，程序以 alice 权限运行。

  7. 链接？

    • 链接可以找到在程序中引用的外部库代码，可以在运行时或编译时完成。
      • 动态链接：首先将其可执行文件加载到内存，动态地使用环境变量找到外部库位置。
      • 静态链接：将使用到的库代码通过编译时的环境直接链接到程序里，可执行文件的大小会非常大。
    • 动态链接成为攻击面的一部分，因为动态链接意味着程序在编译期间未决定部分代码。如果用户可以通过更改环境变量影响丢失的代码，可能会损害程序的完整性。

  8. 环境变量 + 动态链接库的攻击：

    • 原理：在 Linux 系统中，LD_PRELOAD 包含共享库列表。LD_LIBRARY_PATH 包含要搜索的外部库的文件夹。这两个环境变量用户都可以控制。如果某程序为 root Set-UID 程序，会导致普通用户以 root 权限 RCE 的漏洞。

    • 幸运的是，目前的 Linux 系统已经修复了漏洞。即：当运行进程的真实用户 ID 与程序的拥有者的用户 ID 不一致时，进程会忽略掉父进程的 LD_PRELOAD 环境变量；若 ID 一致，则子进程会继承此时运行进程的用户下的 LD_PRELOAD 环境变量，加入共享库。也就是说，父进程被恶意篡改的 LD_PRELOAD 不会对子进程造成影响！

  9. 环境变量 + 外部程序的攻击：

    • 原理：system 函数的权限决定于 EUID。如果包含此函数的 root Set-UID 程序被利用（命令拼接、shellcode 等），可以造成 RCE。

    • execve() 的攻击面更小。因为 system("cmd"); 的原理是 /bin/sh -c 'cmd'。execve() 不调用 shell，而是直接由内核加载并执行指定程序，因此不受环境变量的影响，当在特权程序中调用外部程序时，我们应该使用 execve()。

  10. 环境变量 + 外部库的攻击：

    • 我草我没用 getenv 啊，为啥被打了。
    • 原理：因为有早期 glibc locale 解析漏洞：Set-UID 程序可被 locale 变量触发崩溃或提权。（现在被修复了！）
    • 恶意设置有关 locale 指令调用的环境变量比如 LC_ALL。

CH7 Web 应用攻击

  1. Web 技术由蒂姆·伯纳斯·李（Tim Berners-Lee）提出，1990 年实现最初 Web 服务器、浏览器和 HTTP 协议。

    • Web 架构：Web 服务器、Web 客户端、HTTP 协议。

  2. Web 网页位于 Web 服务器上，用 HTML 语言编写。URL 统一资源定位符，指定 Web 网页在互联网的位置

  3. Web 应用攻击类型：

    • 客户端攻击（攻击用户）：XSS、网络钓鱼、木马
    • 服务器攻击：网页篡改、代码注入攻击、文件操作控制攻击、HTTP 头注入攻击、HTTP 会话攻击等。

XSS 攻击

  1. XSS 攻击是由于 Web 应用程序对用户输入过滤不足而产生的，使得攻击者输入的特定数据变成了 JavaScript 脚本或 HTML 代码。

    • 同源策略：A 网页设置的 Cookie，B 网页能使用这个 Cookie 的条件是：协议相同、域名相同、端口相同。
      • XSS 利用同源策略，达到攻击一个站，影响整个用户（Cookie）的效果。
    • 攻击类型：反射型、存储型、DOM 型。
    • 反射型 XSS：参数型、非持久型跨站脚本。恶意脚本附加到 URL 地址参数中。
    • 存储型 XSS：持久型、对留言、评论、博客等进行攻击。恶意脚本被存储在服务端数据库中。
    • DOM 型 XSS：基于前端 JS、不需要与服务端进行交互。
      • DOM What？是浏览器将 HTML 或 XML 文档解析后形成的一种树形对象模型，它为程序提供了访问、遍历和修改文档结构、内容和样式的标准接口。DOM 给前端设计带来方便，document.URL 即可获取 URL 地址，提取部分内容显示到屏幕上。但是如果提取的部分未经过过滤，显示到屏幕上的东西就可以是 JS 代码，就实施了 XSS。

    • 网络钓鱼：直接注入 HTML 代码等，有更强的隐蔽性。

    • 信息刺探：在客户端执行 JS 代码以访问历史记录、端口信息、剪贴板内容、键盘信息等。

    • 网页挂马：注入木马。

    • 蠕虫：使用 JS Quine 代码传播脚本。

  3. 防范措施：cookie 字段设置参数 HTTP only，对敏感字符进行安全编码。

SQL 注入攻击

  1. SQL 注入定义：向网站提交精心构造的 SQL 语句，导致网站将关键数据返回。

  2. SQL 基础省略。

  3. 万能密码 ' or 1=1 --

  4. 注入类型：字符型、数字型、报错注入、盲注。

  5. SQL 注入流程：寻找注入点、判断数据库类型、猜解表名字段名、登录后台

    • 寻找注入点：

      • 三条语句：id=1，id=1 and 1=1，id=1 and 1=2。

      • 有注入点：前两个语句结果基本相同，第三个语句无回显或找不到记录。

      • 不可注入：语句 1 正常，2 和 3 报错。

    • 判断数据库类型：利用系统变量、banner 等。

    • 猜解表名、字段名：爆破或者盲猜。可以使用 id=1 and 1=(select ....)

    • 登录管理员后台。

  6. SQL 注入危害很大。可获得数据库管理员权限，增删改数据库内的数据，获取敏感信息。执行系统命令提权拿 shell。SQL 注入是从正常的 WWW 端口访问，防火墙一般不报警，很难发现。

    • 提权方法：UNION 合并查询提权、GRANT 提权、xp_cmdshell（OUTFILE）提权、WebShell。

  7. 暴库：通过一些技术手段或者程序漏洞得到数据库的地址，并将数据非法下载到本地。

    • 使用 %5c（ASCII \）作为目录穿越，找到数据库存储位置。
    • 或者使用搜索引擎找到泄露的数据库。

  8. 防范措施：特殊字符转义、输入验证和过滤、参数化方法。

HTTP 会话攻击

  1. CSRF 跨站请求伪造。预测会话 ID，窃取会话 ID，控制会话 ID。
  2. 防范措施：使用编程语言内置的会话管理机制，http only，非会话采纳的环境，不能让会话 ID 长期有效。使用 POST，检查 HTTP referer，验证码，使用 token。

CH8 假消息攻击（1）包嗅探与欺骗

  1. NIC（网络接口卡）是机器和网络之间的物理或逻辑链路。每个 NIC 都有一个 MAC 地址，网络上的每个 NIC 都将监听到线路上的所有帧。NIC 检查每个数据包的目标地址，如果该地址与卡的 MAC 地址匹配，则会进一步复制到内核的缓冲区中；未指定给给定 NIC 的帧将被丢弃。在混杂模式下运行时，NIC 将从网络接收到的每个帧传递给内核，如果嗅探器程序在内核中注册，它将能够看到所有包。在 Wi-Fi 中，被称为监控模式。
  2. BSD 数据包过滤器（BSD Packet Filter，BPF）是一种最早出现在 BSD Unix 系统中的内核级数据包捕获与过滤机制，用于高效地在内核中筛选网络数据包，并将符合条件的数据包交付给用户态程序，不符合条件的数据包丢弃。
  3. 包嗅探：数据包嗅探描述了在实时数据流经网络时捕获这些数据的过程。
    • 使用 pcap API 进行数据包嗅探。
  4. 数据欺骗：伪造数据包中的某些关键信息。
    • 在很多情况下，需要先捕获数据包，然后伪造响应。
    • 流程：先嗅探，保存并改动一些恶意参数到数据包后，进行重放攻击。
  5. 网络通信中的字节序是大端序。

CH9 假消息攻击（2）TCP攻击

  1. TCP 传输控制协议是 Internet 协议套件的核心协议。位于传输层。为应用程序提供主机到主机的通信服务。

    • TCP 较为安全稳定有序，UDP 较低开销，不保证可靠有序。
    • TCP 协议的原理：面向连接，三次握手建立连接，可靠传输，重传，流量控制，拥塞控制。

  2. TCP 服务器程序步骤：

    • 创建套接字
    • 绑定到端口号
    • 监听连接，一旦收到连接请求，将通过三次握手建立连接。已建立的连接放在队列中。
    • 接收连接请求
    • 发送并接收数据。

  3. 一旦建立连接，操作系统在每一端分配两个缓冲区，一个用于发送数据（发送缓冲区）和接收缓冲区（接收缓冲区）。当应用程序需要发送数据时，它会将数据放入 TCP 发送缓冲区。

  4. 发送缓冲区中的每个字节（8bit）在报头中都有一个序列号字段，用于指示数据包的序列。在接收端，这些序列号用于将数据放置在接收缓冲区内的正确位置。一旦数据被放入接收缓冲区，它们就被合并到一个数据流中。

  5. TCP 报头：源端口、目的端口、序列号、确认号、各种标记、窗口大小、checksum

  6. 三次握手：

    • SYN：seq=x
    • SYN ACK seq=y，ack=x+1
    • ACK seq=x+1，ack=y+1

  7. 当服务器收到 SYN 包时，使用 TCB（传输控制块）存储有关连接的信息，这称为半开放连接。

    如果 ACK 没有到达，则会重发 SYN ACK，一段时间后，TCB 将被丢弃。

    • SYN Flooding 原理：SYN Flooding 攻击就是一直向服务器发（随机源地址）SYN 让他的队列填满。
    • 步骤：持续向服务器发送大量 SYN 数据包，通过插入 TCB 记录来消耗队列中的空间。队列填满后，服务就无法访问了。
    • 防御措施：服务器生成一个 SYN Cookie（固定算法 cookie = H(srcIP, srcPort, dstIP, dstPort, time, secret)），作为 SYN-ACK 的初始序列号，不保存 TCB。收到 ACK 之后反算序列号，正确后才建立连接。Flooding 无法找到 Cookie，也就无法攻击成功。

  8. TCP 重置攻击

    • 原理：TCP 一般使用四次挥手（FIN，ACK，FIN，ACK）终止连接，但是也可以直接发 RST 终止连接。

    • 步骤：正确设置源/目的 IP、端口和猜测序列号后，向目标发送大量 RST 消息让他的连接终止。

  9. TCP 会话劫持攻击：

    • 原理：在已建立的 TCP 连接中注入数据，猜测 seq 然后反弹 shell。

    • 步骤：如劫持 telnet 连接，抓包，向内容插入反弹 shell 命令，猜测 seq 后发出。

    • 防御：使攻击者难以伪造数据包：随机端口、初始序列号（对本地攻击无效），加密有效载荷。

CH10 假消息攻击（3）DNS 欺骗

  1. DNS 层次结构：root（根）、TLD（顶级域）、二级域（分配给特定实体）

    DNS 根据区域组织，如 example.com 的域为 *.example.com。

  2. 每个 DNS 区域至少有一个权威域名服务器，提供 DNS 查询的原始和最终答案。可以是主服务器和从服务器。

    主服务器存储所有区域记录的主副本，从服务器从主服务器那里自动更新副本。

  3. root 有 13 个权威域名服务器，是 DNS 查询的起点

  4. 本地 DNS 文件：/etc/hosts 存储 IP-域名键值对，/etc/resolv.conf 存储系统使用的 DNS 服务器和域名解析的基本规则。

  5. 查询过程：

    • 迭代查询。本地 DNS 服务器依次询问 root、TLD、域 DNS 服务器。

    • 递归查询。本地 DNS 服务器询问 root 服务器，root 询问 TLD 服务器，TLD 询问域 DNS 服务器。

  6. DNS 响应：

    • 问题部分：向 nameserver 描述问题
    • 回答部分：回答问题
    • 权威部分：指向权威名称服务器的记录
    • 附加部分：与查询相关的记录

  7. DNS 欺骗：向受害者提供假 IP 地址，诱使他们与不同于他们意图的机器进行通信。

  8. 修改 DNS 文件：获取受害者机器权限后，修改本地 DNS 文件。

  9. 本地 DNS 缓存中毒攻击：在看到来自本地 DNS 的查询后伪造 DNS 应答。

  10. 远程 DNS 缓存中毒攻击：因为无法抓包，需要解决源端口号和 transaction ID 两个随机数的问题。

    • Kaminsky 法：每次构造不同的，域内的 DNS 查询，缓存答案并不重要，在 NS 部分提供答案。

  11. 防范缓存中毒攻击：DNSSEC，TLS/SSL。

  12. 恶意 DNS 服务器的回复伪造攻击：用户访问网站时，会使用网站自带的恶意 DNS 服务器。攻击者使用服务器提供欺诈信息。

    注意：如果在返回包的附加部分和权威部分放一些不属于自己域内的东西，计算机将丢弃这些记录。

    所以这条攻击链应该是构造 RCE 等利用一些漏洞导致的。防范：配置防火墙，判断并过滤不合适的 DNS 包和记录。

  13. Reverse DNS PTR 记录伪造：是指攻击者伪造 PTR 记录的 DNS 响应，使系统在进行 IP → 域名（reverse lookup） 时得到虚假的主机名，从而误导访问控制、身份判断或日志审计，甚至引发进一步攻击。

    • 反向 DNS 控制权为 IP 所有者，比起正向 DNS 的域名所有者来说更加不可信。且正向 DNS 可以 DNSSEC 防护，反向 DNS 安全性更低。反向 DNS 可以轻松监视日志等。

  14. DNS 重新绑定攻击：攻击者利用 DNS 记录可变性，使同一域名在短时间内解析到不同 IP，从而诱导浏览器在同源策略下访问原本不允许访问的内网或本地服务。

    • 不需要服务器漏洞！只利用浏览器和 DNS 的设计缺陷。
    • 防范措施：阻止域名解析到内网 / 本地地址，DNS 结果缓存，校验 Host 等字段，禁止浏览器访问管理接口，配置防火墙规则，拒绝“公网域名 → 私有 IP”解析。

  15. DNS DoS：

    • 攻击 root DNS 服务器很困难，一是高度分布式，大量冗余；二是缓存过期需要 48 小时，见效时间短。
    • TLD 更容易受到攻击。

CH11 熔断与幽灵攻击

  1. 侧信道攻击：利用密码系统实现时泄露的额外信息，推导密码系统中的秘密参数。最近几年，计算错误、执行时间、能量消耗、电磁辐射等侧信道得到深入研究。
  2. 基于缓存的侧信道攻击：对访问数据的时间不同，判断其是否被存在 Cache 里。
    • Flush-Reload 技术：刷新 CPU 缓存，后再加载一些数据到缓存里。
    • 乱序执行：CPU 预测分支，发现分支执行错误后回滚，但是缓存不会被清除。
    • 对策：更换高级 CPU 硬件，使用 KASLR（Kernel Address Space Layout Randomness）内核地址空间布局随机化
  3. 熔断攻击：利用 CPU 在权限检查完成前就执行指令的缺陷，使用户态程序可以读取本应只有内核才能访问的内存数据。
  4. 幽灵攻击：通过误导 CPU 的分支预测，使其在推测执行路径中访问敏感数据，并通过缓存侧信道泄露这些数据。
