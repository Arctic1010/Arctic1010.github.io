网络空间安全技术

网络空间安全技术

CH1 网络空间安全概述

  1. 信息安全基本目标：机密性， 完整性， 可用性

    信息安全就是信息本身的机密性、完整性、可用性的保持。

    • 机密性：防止未经授权使用信息
    • 完整性：防止对信息的非法修改和破坏
    • 可用性：确保及时可靠地使用信息

  2. 网络空间四要素：网络空间载体，网络操作对象，网络活动主体，网络活动形式。

    网络空间的一般性定义：是一种人造的电磁空间，以互联网（空间载体）等为载体，用户（活动主体）在其上对数据（操作对象）进行增删改查等操作，以实现信息通信活动（活动形式）。

  3. 网络空间安全的四层次模型：设备层、系统层、数据层、应用层。

    以及，任何信息系统都会涉及到这四个层次！

    • 设备层安全：信息系统设备对应的安全问题：物理安全、环境安全等。
    • 系统层安全：信息系统自身对应的安全问题：网络安全、软件安全等。
    • 数据层安全：信息系统处理数据带来的问题：数据安全、身份安全等。
    • 应用层安全：信息系统应用过程形成的问题：内容安全、应用安全等。

    网络空间安全是关键信息基础设施保护的必要组成部分，对关键基础设施服务的充分保护有助于满足基础安全需求。四大基础构建模块包括信息安全、应用安全、网络安全、因特网安全。

    五种安全服务：鉴别服务、访问控制服务、数据机密性服务、数据完整性服务、抗抵赖服务。

    八种安全机制：加密、数字签名、数据完整性、访问控制、鉴别交换、通信业务流填充、路由选择控制、公证

CH2 网络空间安全风险及风险管理

  1. 一句话：管理风险 = 了解风险 + 控制风险。

    信息安全风险管理是识别、控制、消除或最小化可能影响系统资源的不确定因素的过程。

  2. 网络空间安全风险术语

    • 资产是任何对组织有价值的东西，是要保护的对象。可以以多种形式存在。
    • 威胁是对信息资产产生的负面影响。
    • 脆弱性是与信息资产有关的弱点或安全隐患。
    • 控制措施是根据安全需求部署的用来防范危险、降低风险的措施。
    • 可能性是指威胁源利用脆弱性造成不良后果的可能性。
    • 影响是指威胁源利用脆弱性造成不良后果的程度大小。
    • 风险是指威胁源采取威胁方式，利用脆弱性造成的不良后果。
    • 残余风险是指采取控制措施后，仍存在的风险。
    • 风险评估是指从风险管理角度，分析系统存在的威胁，评估危害程度。

  3. 常用的四类风险处置方法：减低、转移、规避、接受。

  4. 风险分析：对所需要保护的资产及受到的潜在威胁的鉴别过程。风险是威胁和漏洞的组合。

    公式：R1(A,T,V)=R2(L(T,V),F(Ia,Va))R_1(A,T,V)=R_2(L(T,V),F(Ia,Va))R1​(A,T,V)=R2​(L(T,V),F(Ia,Va))

    • R 表示风险值（Risk）。
    • A 表示资产（Asset），其延伸的 Ia 表示资产价值（Infrastructure）。
    • T 表示威胁（Threat）。
    • V 表示脆弱性（Vulnerability），Va 表示严重程度。
    • L 表示安全事件可能性（Likelihood），F 表示损失（Financial loss）。

  5. 国产化替代替代的是被垄断的国外产品。

    • 国产桌面计算机技术体系对 Wintel 体系的替代（Windows + Intel）
    • 高端服务器和数据库对“IOE”的替代（IBM、ORACLE、EMC^2^）

CH3 信息系统安全保障

  1. 信息系统安全保障是在信息系统的整个生命周期中，从技术、管理、工程和人员等方面提出安全保障要求，确保信息系统的保密性、完整性和可用性，降低安全风险到可接受的程度，从而保障系统实现组织机构的使命。

    简单来讲，在信息系统里确保三要素正常进行，降低安全风险到可接受的程度，保障服务在线。

    • 信息安全意识：认知可能存在的安全问题，明白安全事故对组织的危害，恪守正确的行为方式，清除在安全事故发生时所应采取的措施。
    • 被动攻击：攻击者只监听、收集、分析信息，不对系统资源或通信内容造成任何修改的攻击行为。
    • 主动攻击：攻击者对系统、数据或通信过程进行篡改、注入、伪造或破坏，直接影响系统行为或服务状态的攻击。

  2. 网络空间安全模型是通过建模的思想，精确地、形式化地描述信息系统的安全特性，解释行为，从而解决安全管理问题。可以从模型中开发出关键的描述变量，构建一套安全性评估准则。

  3. PDRR 模型强调自动故障恢复能力：

    保护（Protect）、检测（Detect）、反应（React）、恢复（Restore）。

    其中，如果 Pt >= Dt + Rt，系统安全（保护时间大于等于检测时间加上反应和恢复时间）。

    如果 Pt < Dt + Rt，系统暴露时间 Et 可算出。

  4. 深度防御保障模型：

    信息保障技术框架（Information Assurance Technical Framework，IATF） 是由 NSA 制定的框架。为保护美国政府和工业界的信息与信息技术设施提供技术指南。代表理论为深度防御（Defense in depth）。

    • 强调人、技术、操作这三个核心要素。
    • 关注本地计算环境、区域边界、网络基础设施、支撑基础设施。
    • 在此基础上，对信息系统进行多层防护。

    一句话：信息系统安全不能只依赖单一技术。在设备、系统、应用、数据多层中都部署不同的防御体系，结合对三要素的多种安全机制，即使某几层的体系被攻破，也可以保护系统整体安全。

  5. DDoS 事件：SYN Flood、UDP Flood、ICMP Flood

  6. 常见应急事件：勒索病毒、挖矿木马、远控木马、普通木马

CH4 网络空间安全技术体系与技术

  1. 信息系统安全体系框架：

    信息系统安全的总需求是物理安全、网络安全、信息内容安全、应用系统安全和安全管理的总和，安全的最终目标是确保信息的机密性、完整性、可用性、可控性和抗抵赖性，以及信息系统主体对信息资源的控制。

    完整的信息系统安全体系框架由技术体系、组织机构体系和管理体系共同构建。

    管理体系：法律管理、制度管理和培训管理。

  2. 数据完整性：数据完整性是防止非法实体对交换数据的修改、插入、替换和删除，或者如果被修改、插入、替换和删除时可以被检测出来。数据完整性可以通过消息认证模式来保证。

    完整性机制可以通过密码学、上下文、探测和确认、阻止提供。

  3. 数字签名的特性：不可伪造性、抗抵赖性、保证消息完整性

  4. 公钥加密模型：RSA、ECC

  5. 在网络通信中，可以针对消息内容进行攻击。消息认证做的事情是，对收到的消息进行验证，证明确实是来自声称的发送方，并且没有被修改过。如果在消息中加入时间及顺序信息，则可以完成对时间和顺序的鉴别。

    • 信息加密：用整个消息的密文作为鉴别标识。但是接收方必须识别错误。
    • 哈希函数：公开函数，任意长度消息映射到固定长度散列值。

  6. 增加信息验证码（相同的函数）对数据完整性进行验证。

  7. 公证机制：由第三方公正机构提供，接收通信实体的委托，掌握可供证明的可信赖的信息。可以是仲裁方式或判决方式。

CH5 鉴别机制与技术

  1. 实体鉴别的定义：通过在某一层提供鉴别服务，使其上层实体确信当前通信的对等实体确实是其期望的合法实体，从而防止伪装和冒充攻击。

    对身份鉴别系统的要求

    • 验证者正确识别合法申请者的概率最大化，攻击者伪装申请者让验证者识别成功的概率最小化（可以忽略）。
    • 不具有可传递性，有可证明的安全性，通信有效性。
    • 交互识别，第三方实时参与并可信赖

  2. 实现身份鉴别的途径：基于你所知道的、你所拥有的、个人特征

    • What you know. What you have. What you are.

  3. 基于质询—应答的身份鉴别：由验证者给声称者发送一个确定的值（质询消息），该值参与鉴别信息的运算。产生的非重复质询消息完全由验证者决定，使得每次传输的鉴别信息不同。

    步骤：

    1. A, B shares a key = K

    2. A -> B : ID_A

    3. B -> A : Challenge = n

    4. A -> B : calculate H(F(ID_A, K), n), send T = H(F(ID_A, K), n)

    5. B compares H(F(ID_A, K), n) == T, True or False

CH6 访问控制机制与技术

  1. 访问控制的概念和目标：针对越权使用资源的防御措施。防止对任何资源（如计算资源、通信资源或信息资源）进行未授权的访问，从而使资源在授权范围内使用，决定用户能做什么，也决定代表一定用户利益的程序能做什么。

  2. 访问控制模型分类：

    • 主体：发起访问的实体。客体：被访问的资源。

    • 自主访问控制（DAC）允许客体属主决定主体对该客体的访问权限。可用于商业系统，可灵活调整策略，但是安全性不高。是一种分布式授权管理策略。

    • 强制访问控制（MAC）主体对客体的所有访问请求按照（设定好的）强制访问控制策略进行控制。属主无权决定访问权限。将主体和客体分级，根据主体和客体的级别标记来决定访问模式。

      • 四种强制访问控制的策略：上读、上写、下读、下写。

      • 以经典 BLP 模型 为例（此时级别由保密性决定）：

        • 不上读（No Read Up）：主体不能读取高于自己安全级别的客体
        • 不下写（No Write Down）：主体不能向低于自己安全级别的客体写入信息

        但是可以进行上写下读。主要是防止高密级信息向低密级泄露。

      • Biba 模型保证完整性（级别由完整性决定）

        • 不上写（No Write Up）：低完整性主体不能向高完整性客体写数据，防止不可信数据“污染”高完整性对象。
        • 不下读（No Read Down）：高完整性主体 不能读取 低完整性客体，防止高完整性主体受到低可信信息影响。

        但是可以进行上读下写。

    • 基于角色的访问控制（RBAC）是一种通过角色来管理用户权限的访问控制模型，用户不直接被授予权限，而是通过被分配的角色间接获得权限。

      • 将“用户—权限”的直接关系，转化为“用户—角色—权限”的间接关系。
      • 支持最小权限原则，权限管理清晰集中，易于维护和审计。

CH7 防火墙技术及其发展

  1. 防火墙是在网络间提供安全连接的设备，用于实现和执行网络间通信的安全策略。

  2. 防火墙的种类：

    • 包过滤防火墙：工作在网络层，仅根据数据头的 IP 地址等判断是否通过。
      • 优点：处理速度快，易于配置，对用户透明
      • 缺点：不能识别应用层协议，不能防止 IP spoof，静态策略成为漏洞
    • 应用代理防火墙：工作在应用层，编写不同的代理程序，实现对应用层数据的分析。
      • 优点：可以检查协议特征，对数据包的检测能力很强
      • 缺点：支持的应用数量有限，对用户不透明，性能低
    • 状态监测防火墙：工作在 L2 ~ L4 层，在包过滤的基础上通过规则表和状态表综合分析。
      • 特点：性能高，支持大量应用，在内核级实现检测过滤，通过存储连接状态支持应用层协议检查。
    • 完全内容检测防火墙：工作在 L2 ~ L7 层，状态监测防火墙 + 应用代理防火墙。

  3. 路由器配置：NAT（地址转换）、MAP（端口映射）、IP 与 MAC 绑定。

  4. 防火墙的工作接入方式：路由接入（作为路由器），透明接入（作为交换机），混合接入（同时作为路由器和交换机）。

    • 路由接入：防火墙作为三层设备（具备路由功能），流量必须经过防火墙，需要重新规划 IP 地址和路由。

      • 优点：网络结构清晰，易于与 NAT、VPN 等结合。
      • 缺点：需改动网络结构，对原有配置影响大，切换风险较高。

    • 透明接入：防火墙作为二层设备，对网络透明，不改变路由结构。

      • 优点：部署简单，适合快速上线
      • 缺点：不具备路由功能，可扩展性差，不支持 NAT 等。

    • 混合接入：防火墙同时支持三层和二层工作模式，灵活适配复杂网络环境。

      • 优点：兼具安全性和部署灵活性
      • 缺点：配置复杂，结构复杂，运维难度高。

    • 企业防火墙的典型部署：DMZ（Demilitarized Zone）隔离区。

  5. 衡量防火墙性能的五大指标：吞吐量、时延、丢包率、背靠背、并发连接数。

    • 吞吐量：防火墙一秒能收到并转发多少数据。
    • 时延：数据包通过防火墙需要增加多少处理时间。
    • 丢包率：工作过程中未能成功转发的数据包比例。
    • 背靠背：防火墙在不发生丢包的情况下，能够连续处理的最大数据包数量。
    • 并发连接数：防火墙同时能够维护的最大会话数量。

CH8 入侵检测

  1. 入侵（Intrusion）是指在非法或未经授权的情况下，试图存取或处理系统或网络中的信息，或破坏系统或网络正常运行，致使系统或网络的机密性、完整性和可用性受到破坏的故意行为。

  2. 入侵检测模型（IDM）：

    • 通用入侵检测模型（Denning 模型）
      • 通过对系统和网络行为进行统计分析，发现偏离正常行为模式的异常，从而检测入侵行为。
    • 层次化入侵检测模型（IDM）
      • 将 IDS 系统按功能和层次分为多个级别。结构清晰可扩展但是系统复杂，需考虑层间通信成本。
    • 管理式入侵检测模型（SNMP-IDSM）
      • 将入侵检测系统纳入网络管理体系，适合大型网络运维。

  3. 入侵检测系统：CIDF 架构：入侵检测系统的通用模型框架

    • 是一种用于规范入侵检测系统（IDS）组成、接口和协作方式的体系结构模型，目的是使不同厂商、不同类型的 IDS 组件能够协同工作和互操作。
    • 特点：模块化、可扩展、互操作性强。
    • 核心组件：事件生成器（采集原始数据），事件分析器（识别入侵行为），事件数据库（存储事件），响应单元（执行防御）。

  4. 入侵监测系统（Intrusion Detection System，IDS）分析技术：异常检测、特征检测。

    • 异常检测：基于行为的检测，标识出不符合正常模式的活动，实现难度较大，因为难以明确划分正常模式和异常模式。
    • 特征检测：也称为误用检测，建立入侵行为模型（攻击特征库），判断当前行为是否符合已知的攻击模式。现已被广泛使用。
      • 使用的检测方法：基于专家系统、简单规则的模式匹配、攻击签名分析、统计模式匹配、状态转换分析等

  5. IDS 的两个指标：漏报率、误报率。

    IDS 的缺点：IDS 是旁路部署，事后发现危险，实时性不强。无法分析加密数据。只能检测已知攻击（0 Day 等不好使）。带宽要求高。攻击响应依赖人工。

CH9 其他网络安全技术及设备

  1. 安全隔离与信息交换系统（网闸）

    • 应对网络安全漏洞传统解决方案为：物理隔离。

      • 不连接，没有连接，就能防止一切直接网络攻击。

      • 人工拷盘可以解决信息传输问题。

      • 缺点：延时长速度慢，可靠性低，人工操作风险性大。

    • 网闸是什么？网闸是在两个不同安全域（隔离程度高的网络）之间，通过协议转换的手段，以信息摆渡的方式实现安全的数据交换，且只有被系统明确要求传输的信息才可以通过。

    • 产品定位：安全隔离网闸的思路是在保证必须安全的前提下，尽可能互联互通。与防火墙不同。防火墙的思路是在保障互联互通的前提下，尽可能安全。

    • 优势：在不可信网络牺牲自己来有效应对攻击，充分保护可信网络。

  2. 网络准入控制（NAC）

    • 控制终端设备是否、以及以何种权限接入网络的安全机制。
    • 核心功能：身份鉴别，终端安全状态评估，准入策略控制，隔离与整改，更新与修复。
    • NAC 用户流程：用户尝试访问一个网页或客户端，用户输入用户名和密码，服务器执行设备和网络扫描，评估安全漏洞。如果设备不符合安全策略或登录信息不正确，拒绝用户接入；如果设备是清洁的，接入网络。

  3. 入侵防御系统（IPS）

    • IPS 是一种集入侵检测和防御于一体的安全产品，它不但能检测入侵的发生，而且能通过一定的响应方式，实时地中止入侵行为的发生和发展，实时地保护信息系统不受实质性的攻击。
    • IPS 就是防火墙 + 入侵检测系统。
    • 功能：识别并阻断攻击，向管理控制台发送日志信息，集成防火墙的过滤功能。

  4. IDS 和 IPS 的区别：

    IDS 被动检测，旁路部署，通过镜像获取数据，不阻断流量，对网络几乎没有影响，实时性较弱，无漏报。

    IPS 主动检测，串联部署，可能影响性能，阻断流量，无误报，实时性较强。

CH10 虚拟专用网技术

  1. VPN 是一种在公共网络上通过隧道、加密和认证技术构建安全通信通道的技术。

    • 主要协议：IPSec、SSL/TLS
    • IPSec：工作在网络层（IP 层），对应用透明，安全性高。
      • IP 鉴别头 AH 机制：通过将 IP 数据包插入一个 AH 头来提供安全服务。AH 能为 IP 数据包提供完整性验证和身份认证。
        • AH 传输模式：【原 IP 头】【AH 头】【TCP/UDP 头】【数据】
        • AH 隧道模式：【新 IP 头（协议字段为 51）】【AH 头】【原 IP 头】【TCP/UDP 头】【数据】
      • IP 封装安全载荷 ESP：ESP 将整个 IP 头或上层协议（传输层协议）部分封装到一个 ESP 载荷之中，然后做加密处理，只鉴别 ESP 头以后的信息，比 AH 鉴别的范围窄。
        • ESP 传输模式：【原 IP 头】【ESP 头】〖【TCP/UDP 头】【数据】〗【ESP 尾】
        • ESP 隧道模式：【新 IP 头（协议字段为 50）】【ESP 头】〖【原 IP 头】【TCP/UDP 头】【数据】〗【ESP 尾】
      • 为什么叫隧道模式？可以看到，原来的包被 AH 或 ESP 完全保护，像隧道一样，而单从外面的结构是看不出来的。隧道模式可以防止内网 IP 地址暴露，支持子网互联。
      • IPSec 采用 IKE 完成 SA 的协商。
    • SSL：Secure Socket Layer 安全套接层，工作在应用层。
      • 提供的服务：认证用户和服务器，加密数据，维护数据的完整性。
      • 技术栈：SSL 握手协议/HTTP、SSL 记录协议、TCP、IP、…
      • SSL 握手协议：建立安全连接，服务器和客户机相互验证和密钥交换。
      • SSL 记录协议：为 SSL 提供了机密性和消息完整性。
        • 应用实例：HTTPS 结合 HTTP 和 SSL 来实现 Web 服务器和浏览器的安全通信，单向鉴别。
    • TLS：Transport Layer Security 传输层安全，

  2. VPN 通道的建立方式：

    • Host to Host 模式：要求两边主机都要支持 IPSec，VPN 网关可以支持可以不支持 IPSec。

      • Host A ===== 加密隧道 ===== Host B

      • 适用于两台主机点对点通信，内部系统之间的安全连接。

    • Host to VPN 模式：Host 必须支持 IPSec，VPN 网关必须支持 IPSec。

      • Host A ===== 加密隧道 ===== VPN Gateway ── 内网
      • 适用于单台主机访问某个内部网络。

    • VPN to VPN 模式：主机可以不支持 IPSec，VPN 网关必须支持 IPSec。

      • LAN A ── VPN Gateway A ===== 隧道 ===== VPN Gateway B ── LAN B
      • 使用 IPSec 隧道模式，封装整个 IP 包，隐藏内部网络结构。适用于站点到站点的 VPN。

    • Remote User to VPN 模式：用户通过公网远程接入内网。VPN 网关必须支持 IPSec。

      • Remote User(Laptop) === 隧道 === VPN Gateway —— 内网
      • 适用于远程用户访问内网的场景，需要严格身份认证。

CH11 系统安全

  1. 系统安全的定义：采取各种措施和实践，以保护计算机系统及其相关资源不受未经授权的访问、使用、披露、破坏、修改或者中断。
  2. 系统安全的核心目标：确保信息的机密性、完整性、可用性，防止恶意攻击和意外事故损害系统。
    • 操作系统安全：内存隔离、堆栈保护、数据执行保护，文件系统权限
    • 数据库安全：SQL 注入，XSS，配置错误
  3. 备份：全备份、增量备份、差异备份
  4. 漏洞分析：静态、动态分析，模糊测试
